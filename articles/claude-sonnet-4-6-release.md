## 何が起きたのか

2026年2月17日、AnthropicがClaude Sonnet 4.6をリリースしました。これは単なるマイナーアップデートではありません。**Sonnetクラスのモデルが、初めてOpusクラスに肉薄した**という、階級を超えた進化です。

しかも、価格はSonnet 4.5と同じ。つまり、**Opus級の性能を、Sonnetの価格で**手に入れられるようになった、ということです。

この記事では、ベンチマーク・コスト・実用面から、Sonnet 4.6が何を変えたのかを整理します。

---

## 数字で見る進化

### コーディング性能

**SWE-bench Verified**（実際のGitHub issueを解決するタスク）

| モデル | スコア |
|--------|--------|
| Sonnet 4.6 | 79.6% |
| Opus 4.6 | 80.8% |
| Sonnet 4.5 | 64.1% (推定) |

**差はわずか1.2ポイント**。これは、コーディングタスクにおいて、SonnetがOpusとほぼ同等になったことを意味します。

### 開発者の評価（Claude Code内テスト）

- **70%の開発者がSonnet 4.5より好む**
- **59%の開発者がOpus 4.5より好む**

「より好む」の理由：
- 過剰設計（overengineering）が少ない
- "怠け（laziness）"が減った
- 指示追従が改善
- 嘘の成功報告（false claims of success）が減少
- ハルシネーションが減少
- マルチステップタスクでの一貫性が向上

**つまり、Opusより性能が低いはずのSonnetが、実用面ではOpusを上回る評価を得ている**ということです。

### エージェント性能（コンピューター使用）

**OSWorld-Verified**（ChromeやLibreOffice、VS Codeを実際に操作するタスク）

| モデル | スコア |
|--------|--------|
| Sonnet 4.6 | 72.5% |
| Opus 4.6 | 72.7% |
| Sonnet 4.5 | 53.0% (推定) |

保険業界の複雑なフォーム入力タスクでは、**Sonnet 4.6が94%を達成**（過去最高）。

### 推論性能

**ARC-AGI-2**（高度な推論タスク）

- Sonnet 4.6（high effort mode）: **60.4%**
- これはフロンティアレベル（最先端）

**GPQA Diamond**（PhDレベルの科学問題）

- Sonnet 4.6: **89.9%**

### ドキュメント理解

**OfficeQA**（企業文書から事実を抽出して推論）

- Sonnet 4.6: Opus 4.6と**同等**

## コストパフォーマンスの革命

### 価格据え置き

| モデル | 入力 | 出力 |
|--------|------|------|
| Sonnet 4.6 | $3 / 1M tokens | $15 / 1M tokens |
| Sonnet 4.5 | $3 / 1M tokens | $15 / 1M tokens |
| Opus 4.6 | $15 / 1M tokens | $75 / 1M tokens |

**Opusの1/5の価格で、Opusとほぼ同等の性能**。

これは、「Opusは高すぎて常用できない」と感じていたユーザーにとって、大きな転換点です。

### 実例：100万トークン処理した場合

あるコードベース全体（約100万トークン）を読み込んで、バグ修正を依頼する場合：

- **Opus 4.6**: 入力 $15 + 出力（仮に10万トークン）$7.5 = **$22.5**
- **Sonnet 4.6**: 入力 $3 + 出力 $1.5 = **$4.5**

**約5倍の差**。これは、毎日使うか、週1回使うかの差になります。

## 実際の改善点

### 1. コーディング

- **一貫性の向上**: 変更前のコンテキストをより正確に読む
- **共通ロジックの統合**: 重複コードを減らし、保守性の高いコードを生成
- **バグ検出**: Opus 4.6との差が縮まり、並列でレビュワーを走らせやすくなった

**ユーザーの声**（公式発表より引用）:

> "Sonnet 4.6は、深いコードベース作業において、かつては高価なモデルを必要としていたタスクのデフォルト選択肢になりつつある。"

> "iOSコード生成において、仕様準拠、アーキテクチャ、そして我々が求めていない最新ツールの採用まで、全てが一発で決まった。正直、驚いた。"

### 2. エージェント（コンピューター使用）

- **人間レベルの操作**: 複雑なスプレッドシート、複数タブにまたがるWebフォーム入力
- **16ヶ月間の進化**: OSWorldスコアが大幅に向上（初期バージョンからの累積的改善）
- **プロンプトインジェクション耐性**: Sonnet 4.5から大幅に改善、Opus 4.6と同等

### 3. 長文推論

- **1M トークンコンテキスト**（ベータ版）
  - コードベース全体
  - 長大な契約書
  - 数十本の研究論文
  
- **長期戦略**: Vending-Bench Arena（ビジネスシミュレーション）で、最初の10ヶ月は投資に集中し、最終段階で収益化にピボットする戦略を自律的に編み出した

### 4. フロントエンド・デザイン

**複数のユーザーが独立に報告**:

- レイアウトが洗練されている
- アニメーションがスムーズ
- デザイン感覚が向上
- 本番品質までのイテレーション回数が減少

## 新機能

### 1. ハイブリッド推論（Adaptive Thinking）

- **適応的思考**: タスクに応じて、即座に応答 or ステップバイステップ思考を選択
- **Extended Thinking**: 深い推論が必要な場合に明示的に有効化
- API経由でeffortパラメータによる細かい制御が可能

### 2. コンテキスト圧縮（Context Compaction）

- 会話が長くなると、古いコンテキストを自動的に要約
- **実効的なコンテキスト長が1Mを超える**場合もある

### 3. Web検索・フェッチの自動最適化

- 検索結果を**自動的にコード実行してフィルタリング**
- 関連性の高いコンテンツのみをコンテキストに保持
- トークン効率 ＋ 応答品質の向上

## 誰がどう使うべきか

### Sonnet 4.6が最適なケース

- **毎日使うコーディングタスク**
  - バグ修正
  - 機能追加
  - リファクタリング
  - コードレビュー並列実行

- **エージェント的な作業**
  - ブラウザ操作
  - データ入力
  - 複数ツールの連携

- **長文ドキュメントの処理**
  - 契約書レビュー
  - 研究論文の要約
  - 技術仕様書の分析

- **フロントエンド開発**
  - UIコンポーネント生成
  - レスポンシブデザイン
  - アニメーション実装

### まだOpus 4.6が必要なケース

Anthropic公式の推奨（発表文より）:

> "Opus 4.6は、最も深い推論を要するタスクにおいて、依然として最強の選択肢です。例えば、コードベース全体のリファクタリング、複数エージェントのワークフロー調整、正確性が最優先される問題などです。"

つまり：
- **コードベース全体の大規模リファクタリング**
- **複数エージェントの協調制御**
- **「正しさが全て」の問題**（医療、金融、法務など）

## 実用的な使い分けガイド

### パターン1: デフォルトはSonnet

```
日常のコーディング → Sonnet 4.6
  ↓（行き詰まったら）
深い推論が必要な局面 → Opus 4.6
```

**コスト削減効果**: 90%のタスクがSonnetで完結すれば、コストは1/5近くに。

### パターン2: 並列レビュー

```
実装: Sonnet 4.6
  ↓
レビュー1: Sonnet 4.6
レビュー2: Sonnet 4.6
レビュー3: Sonnet 4.6
  ↓
最終チェック: Opus 4.6
```

Sonnetのコストパフォーマンスにより、**複数並列でレビューを走らせてもコストが抑えられる**。

### パターン3: フェーズ分け

```
プロトタイピング: Sonnet 4.6（速度重視）
  ↓
本実装: Sonnet 4.6（80%のケース）
  ↓
クリティカル部分: Opus 4.6（正確性重視）
```

## 他社モデルとの比較

### OpenAI GPT-4 Turbo

- コーディング: SWE-benchでSonnet 4.6が大幅にリード
- コスト: GPT-4 Turbo ($10/$30) vs Sonnet 4.6 ($3/$15) → **Sonnetが1/2**

### Google Gemini 1.5 Pro

- 長文コンテキスト: Gemini 1.5 Proは2M対応、Sonnet 4.6は1M（ただしコンテキスト圧縮で実効的に拡張）
- エージェント: OSWorldではSonnet 4.6がリード
- コスト: Gemini 1.5 Pro ($3.50/$10.50) vs Sonnet 4.6 ($3/$15) → **入力は同等、出力はGeminiが安い**

### DeepSeek V3

- コスト: DeepSeek V3は超低価格（$0.27/$1.10）
- 性能: コーディングではSonnet 4.6が上回るが、コスト制約が厳しい場合はDeepSeekも選択肢

**結論**: Sonnet 4.6は、**性能とコストのバランスが最も優れている**。

## まとめ: 階級を超えた進化

Sonnet 4.6は、**Sonnetという階級の枠を超えました**。

これまで：
- Sonnet = 日常使い
- Opus = 本気モード

これから：
- Sonnet 4.6 = 日常使い ＋ 本気モード（90%）
- Opus 4.6 = 本気モード（残り10%）

これは、「Opusは高くて常用できない」と感じていたユーザーにとって、**フロンティアモデルが日常に降りてきた瞬間**です。

### AIモデルの進化スピード

2024年10月にコンピューター使用が「実験的」としてリリースされた時、Anthropicは「不器用でエラーも多いが、急速に改善する」と書きました。

16ヶ月後、OSWorldスコアは大幅に向上し、保険業界のタスクでは94%を達成。これは、**AIモデルの進化が、人間の予想を超えて加速している**ことを示しています。

Sonnet 4.6は、その加速の象徴です。
